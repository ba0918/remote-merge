# Project Review Rules

plan-reviewer がこのプロジェクトの計画をレビューする際に適用する追加ルール。
（generate-review-rules スキルにより自動生成。手動で調整可能）

## Architecture

CLAUDE.md Design Principles より:

- **レイヤー構造**: ドメイン層(app/) → サービス層(runtime/) → ハンドラ層(handler/) → UI層(ui/)
- **依存方向**: 上位レイヤーから下位レイヤーへの一方向のみ許可
- **責務分離**: 責務が混在する場所にビジネスロジックを書かない。上位モジュールはドメインロジックの組み合わせだけで構成する
- **グルーコード禁止**: 接続・ルーティングにビジネスロジックを混ぜない
- **モジュール分割**: 行数を抑えることが目的ではなく、責務を混在させないことが目的。200行は目安であり兆候
- **ドメイン層は純粋関数**: 副作用なし。入出力だけでテストできる
- **ハンドラ層は薄い変換**: イベント → サービス呼び出しの変換のみ
- **UI層は描画のみ**

## Security

- SSH 接続情報（鍵パス、パスフレーズ）のログ出力禁止
- リモートパスのパストラバーサル検証必須（validate_path で絶対パス拒否）
- shell_escape によるコマンドインジェクション防御（SSH exec 経由のコマンド構築時）
- sensitive ファイル（.env, *.pem 等）のマージ/diff 前に警告表示
- optimistic locking: マージ前にリモート mtime を再チェック

## Performance

- SSH exec 回数の最小化（バッチ処理を優先。batch_read チャンク分割）
- 大量ファイル処理時のメモリ使用量（10万ファイル規模を想定。ストリーミング処理を検討）
- async/await の適切な使用（tokio ランタイム。不要な Arc/Mutex の増殖を防ぐ）
- 遅延読み込み: ディレクトリ内容は展開時、ファイル内容はオンデマンドで取得
- ARG_MAX 制約への対応（レガシー環境では 128KB。コマンド引数のチャンク分割）

## Language/Framework Specific

### Rust

- `unsafe` の使用は原則禁止（使う場合は理由をコメントで明記）
- `unwrap()` は本番コードで使用禁止（テストコードは許可）
- エラーハンドリングは `anyhow` の `.context()` で文脈を付与
- clippy 警告ゼロを維持（pre-commit hook で強制）
- `cargo fmt` 準拠（pre-commit hook で強制）

### russh / SSH

- SSH 切断時の自動再接続は行わない（安全性のため意図的な設計）
- シンボリックリンクはリンクターゲットパスで比較（デリファレンスしない）
- バイナリファイルは SHA-256 ハッシュ比較（コンテンツ diff ではない）
- CentOS 5 等レガシー OpenSSH（4.3）との互換性考慮（ed25519 未対応、レガシー Kex のみ）

### ratatui / TUI

- UI 文言は全て英語統一（ダイアログ、ステータスメッセージ、エラー、CLI ヘルプ）
- コードコメントは日本語 OK

## Testing

- **コード変更時は必ずテストを書く**（テストなしのコミットは禁止）
- テストが書けない場合は設計が悪い。ロジックを純粋関数に切り出してテスト可能にする
- テストは各モジュール末尾の `#[cfg(test)] mod tests` に配置（Rust 慣習）
- ドメインロジックは入出力だけでテストできる純粋関数にする
- pre-commit hook: `cargo fmt --check` + `cargo clippy`
